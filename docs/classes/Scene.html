Scene Class

Extends Component
Defined in: src/scene.js:1
Module: XEO

A Scene models a 3D scene as a fully-editable and serializable component-object graph.

Scene Structure

A Scene contains a soup of instances of various Component subtypes, such as GameObject, Camera, Material, Lights etc. Each GameObject has a link to one of each of the other types, and the same component instances can be shared among many GameObjects.

Under the hood: Within xeoEngine, each GameObject represents a draw call, while its components define all the WebGL state that will be bound for that call. To render a Scene, xeoEngine traverses the graph to bind the states and make the draw calls, while using many optimizations for efficiency (eg. draw list caching and GL state sorting).

Default Components

A Scene provides its own default flyweight instance of each component type (except for GameObject). Each GameObject you create will implicitly link to a default instance for each type of component that you don't explicitly link it to. For example, when you create a GameObject without a Lights, the GameObject will link to the Scene's default lights. This mechanism provides training wheels to help you learn the API, and also helps keep examples simple, where many of the examples in this documentation are implicitly using those defaults when they are not central to discussion.

At the bottom of the diagram above, the blue Material, Geometry and Camera components represent some of the defaults provided by our Scene. For brevity, the diagram only shows those three types of component (there are actually around two dozen).

Note that we did not link the second GameObject to a Material, causing it to be implicitly linked to our Scene's default Material. That Material is the only default our GameObjects are falling back on in this example, with other default component types, such as the Geometry and the Camera, hanging around dormant until a GameObject is linked to them.

Note also how the same Camera is linked to both of our GameObjects. Whenever we update that Camera, it's going to affect both of those GameObjects in one shot. Think of the defaults as the Scene's global component instances, which you may optionally override on a per-GameObject basis with your own component instances. In many Scenes, for example, you might not even bother to create your own Camera and just let all your GameObjects fall back on the default one.

Example

Here's the JavaScript for the diagram above. As mentioned earlier, note that we only provide components for our GameObjects when we need to override the default components that the Scene would have provided them, and that the same component instances may be shared among multiple Objects.

var scene = new XEO.Scene({
      id: "myScene"   // ID is optional on all components
 });

var material = new XEO.PhongMaterial(myScene, {
      id: "myMaterial",         // We'll use this ID to show how to find components by ID
      diffuse: [ 0.6, 0.6, 0.7 ],
      specular: [ 1.0, 1.0, 1.0 ]
  });

var geometry = new XEO.Geometry(myScene, {
      primitive: "triangles",
      positions: [...],
      normals: [...],
      uvs: [...],
      indices: [...]
 });

var camera = new XEO.Camera(myScene);

var object1 = new XEO.GameObject(myScene, {
      material: myMaterial,
      geometry: myGeometry,
      camera: myCamera
 });

// Second object uses Scene's default Material
var object3 = new XEO.GameObject(myScene, {
      geometry: myGeometry,
      camera: myCamera
 });

Finding Scenes and Components by ID

We can have as many Scenes as we want, and can find them by ID on the XEO object's scenes map:

var theScene = XEO.scenes["myScene"];

Likewise we can find a Scene's components within the Scene itself, such as the Material we created earlier:

var theMaterial = myScene.components["myMaterial"];

The Default Scene

When you create components without specifying a Scene for them, xeoEngine will put them in its default Scene.

For example:

var material2 = new XEO.PhongMaterial({
   diffuse: { r: 0.6, g: 0.6, b: 0.7 },
   specular: { 1.0, 1.0, 1.0 }
});

var geometry2 = new XEO.Geometry({
    primitive: "triangles",
    positions: [...],
    normals: [...],
    uvs: [...],
    indices: [...]
});

var camera = new XEO.Camera();

var object1 = new XEO.GameObject({
    material: material2,
    geometry: geometry2,
    camera: camera2
});

You can then obtain the default Scene from the XEO object's scene property:

var theScene = XEO.scene;

or from one of the components we just created:

var theScene = material2.scene;

Note: xeoEngine creates the default Scene as soon as you either create your first Sceneless GameObject or reference the XEO object's scene property. Expect to see the HTML canvas for the default Scene magically appear in the page when you do that.

Saving and Loading Scenes

The entire runtime state of a Scene can be serialized and deserialized to and from JSON. This means you can create a Scene, then save it and restore it again to exactly how it was when you saved it.

// Serialize the scene to JSON
var json = myScene.json;

// Create another scene from that JSON, in a fresh canvas:
var myOtherScene = new XEO.Scene({
     json: json
 });

Note: this will save your Geometrys' array properties (positions, normals, indices etc) as JSON arrays, which may stress your browser if those arrays are huge.
